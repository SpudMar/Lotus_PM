8. Risks and How We Manage Them

RiskLikelihoodHow We Manage It
Development takes longer than planned Medium Phased approach. Each phase delivers usable value independently. Not all-or-nothing.
Hit a technical wall Low Modular design. We can swap components without rewriting everything. No bad decisions are permanent.
Learning as I build Reality AI-assisted development with Claude Code changes what one person can build. The tools are the mitigation.
NDIS rules change Medium Modular design. Rule changes are isolated updates, not system rewrites. Same as any compliant software.
Staff resist the change Low Staff are involved in testing from Day 1. Their feedback shapes the product. They see it being built for them.
Entiprius improves significantly Low Their legacy architecture has fundamental limits. They can patch, not rebuild. We're building from scratch.

What We Are Not Doing

  • We are not going all-in overnight ‚Äî the pilot runs alongside Entiprius
  • We are not skipping testing ‚Äî every module is tested before it touches real data
  • We are not rushing the migration ‚Äî clients move over when the system is solid, not before
  • We are not building something fragile ‚Äî modular, well-tested, professionally structured
